# 闲鱼工作台自动化 · 使用说明书

> 一句话：用浏览器接管你的闲鱼卖家工作台，**按商品自动切换 AI 身份**回复买家，
> 每个客户独立记忆、互不串台，支持报价策略、商品采集、人工接管等。

## 一、它能做什么

- **按商品切换身份**：不同商品链接对应不同"卖家身份"（人设 + 目标 + 报价），AI 自动判断当前对话属于哪个商品，用对应身份回复。
- **每客户独立记忆**：记住每个买家说过什么、需求是什么，A 客户的需求不会串到 B 客户。
- **不回错人**：多个买家同时来消息时，精准回到对应的对话框，不会群发错乱。
- **目标驱动**：比如"网页代做"身份会主动收集客户需求（项目/用途/样式…），集齐后生成需求文档；"答疑型"身份解答疑问点到为止。
- **报价策略**：按产品类目设置价格区间、低预算话术、加售引导，AI 报价更靠谱。
- **商品采集 + AI 生成身份**：贴一个商品链接，自动抓商品信息并由 AI 生成对应身份。
- **人工接管**：随时一键暂停某个客户的 AI，自己接手聊。

## 二、整体结构（先理解这个）

这是**两个程序**，通过同一个数据库协作：

| 程序 | 作用 | 怎么启动 |
|---|---|---|
| **后端**（管理后台） | 提供网页后台，管身份/记忆/日志等 | `start.bat` 自动起 |
| **Worker**（自动回复机） | 接管 Edge、监听消息、自动回复 | 后台「上线运行」或 `start.bat` 自动上线 |

- **后台地址**：http://127.0.0.1:8090
- **Worker** 跑起来后会接管一个带调试端口的 Edge 浏览器，盯着你的闲鱼工作台 IM 页面收发消息。
- **关键**：只有 Worker「上线运行」时，才会自动回复买家。后台只是管理界面，开着后台 ≠ 在自动回复。

## 三、快速开始（每天怎么用）

### 开工
**双击 `start.bat`**（项目根目录），它会自动：
1. 清掉上次残留的进程（防止重复回消息）
2. 启动后端
3. 自动让 Worker 上线（首次会拉起调试 Edge，约 10 秒）
4. 打开后台网页

### 收工
**双击 `stop.bat`**，停掉后端和 Worker。

### 改了配置/代码之后
先 `stop.bat` 再 `start.bat`（或直接 `start.bat`，它会先杀旧的）。
> 只改了网页界面（看起来不一样）不用重启，浏览器按 **Ctrl + F5** 强刷即可。

## 四、前置条件

1. **Edge 已登录闲鱼工作台**。Worker 用的是项目自带的调试 Edge（`edge_debug` 这个独立配置），第一次需要在它里面登录一次闲鱼，之后会记住登录态。
2. **config.yml 里填了 DeepSeek 的 API Key**（`ai.api_key`）。这是 AI 回复的钥匙。
3. 网络能访问 `api.deepseek.com`。

## 五、各功能详解

后台左侧分四组：**运营 / AI 配置 / 工具 / 系统**。

### 1. 概览

- **自动回复 Worker**：
  - 「▶ 上线运行」：启动自动回复（首次会自动拉起调试 Edge）。
  - 「■ 关闭」：停止自动回复。
  - 状态显示：`运行中(心跳 N 秒前)` = 正常；`掉线/心跳异常` = 系统会自动重启；`连续重启失败` = 需要你看日志手动处理。
- **统计卡**：启用身份数、客户数、今日回复、累计回复。
- 状态每 15 秒自动刷新。

> **心跳/自动重启**：Worker 每 30 秒上报一次"我还活着"。后端每分钟检查，发现掉线会自动把它拉回来（仅当你点过「上线运行」时才会自动重启；没点过绝不会自己启动）。

### 2. 身份管理（重点）

这是核心。一个"身份"= 在什么情况下用 + 什么人设 + 什么目标 + （收集型的）要问哪些信息。

**A. 🤖 贴链接，AI 自动生成身份**（最省事）
- 把闲鱼商品链接粘进去 → 点「AI 生成身份」。
- 系统会：抓商品信息（标题/价格/详情）→ 让 AI 判断这是服务类还是实物类 → 自动生成人设、目标、槽位，**预填到下方表单**。
- 你检查一下、改改，点「保存身份」。

**B. 手动新建 / 编辑身份**
- **命中条件**（决定这条对话用哪个身份）：
  - **关键词**：买家消息里含其一就用此身份（如 `网页,网站,建站`）。
  - **商品 ID**：最精准。闲鱼消息里自带商品 ID，填了就直接命中。
  - **链接关键词**：支持 `*` 通配、`re:` 正则。
- **人设**：AI 以什么身份、口吻回复。
- **目标类型**：
  - **无目标**：只按人设回复。
  - **收集需求型（collect）**：配几个"关键信息点（槽位）"，AI 只追问还没问到的，**集齐后停止追问并生成需求文档**（在客户记忆里看）。适合网页代做、定制开发这类要先了解需求的。
  - **答疑型（answer）**：解答疑问、促成交易，点到为止。适合实物/虚拟商品。
- **优先级**：多个身份都可能命中时，数字大的优先。

> **会话锁定**：一段对话第一次命中某身份后会**锁定，整轮沿用**。买家后面就算说了别的商品的关键词，也不会中途跳身份。

### 3. 报价策略

按"产品类目"配报价话术，AI 回复涉及价格时会自动套用。

- **命中关键词**：买家消息含它、**或当前身份名含它**，就套用此报价策略（所以整段对话稳定，不会因为某句没提关键词就丢）。
- **参考报价区间**：如 `50-300元` / `8.8元起`。
- **基础话术**：报价原则、适合的需求范围、语气。
- **低预算处理**：仅当买家说"便宜/预算有限/低"等时，AI 才追加这段。
- **加售引导**：建议客户从基础版开始、后续升级等。

> 预置了三类示例（小程序 / 网页定制 / 自动化脚本），可直接改或删。改完在「AI 试聊」里选对应身份聊"多少钱"就能验证效果。

### 4. 客户记忆

每个客户（按会话）一份独立记忆，AI 据此记住其需求、互不串台。

- **列表**：客户、商品、身份、需求摘要、目标进度、**未读数**、更新时间。
- **暂停 AI / 恢复 AI（人工接管）**：对某客户点「暂停AI」，AI 就**不再自动回复他**，只记录消息、累计未读，你自己接手聊；想交回 AI 点「恢复AI」。暂停中的客户那一行会标黄 + 标"人工接管"。
- **查看**：看完整对话、AI 维护的需求摘要、关键信息槽位、以及（收集型集齐后）生成的**需求文档**。
- **编辑**：摘要和槽位可手动改（AI 偶尔抽取有误时兜底）。
- **清空**：清掉 AI 维护的摘要/槽位（对话记录保留）。

### 5. AI 试聊

不发送到闲鱼，纯测试 AI 会怎么回。

- 选一个身份（或留空让它按商品/关键词自动判断）。
- 像买家一样连续发消息，看 AI 回复。
- 右侧实时显示这次会话的记忆 / 槽位进度 / 需求文档。
- 换商品测试前先点「🗑 重置对话」。

> 调身份人设、目标、报价话术后，都建议先来这里试聊验证，再用到真实回复。

### 6. 回复日志

每条自动回复的记录：时间、买家消息、回复内容、**来源**、是否已发。

**来源标签**：
- **`ai`**（绿）= AI 正常生成的真实回复 ✓
- **`fallback`**（橙）= AI 那次**调用失败了**，退回用固定兜底话术（config 里的 `fallback_reply`）。偶尔一两条多半是 DeepSeek 网络波动；如果频繁出现，需要排查（看 `logs/worker.log`）。

### 7. 设置

- **默认人设**：未命中任何身份时的兜底人设。
- **最终注入 AI 的 system prompt（预览）**：能看到 AI 实际收到的设定长什么样。
- **当前配置（只读）**：展示 config.yml 里的关键配置（回复延迟、营业时段、每日上限等）。改这些要编辑 `config.yml` 后重启后端。

## 六、商品采集

把商品的标题/价格/图片/卖家抓下来，用于快速配置身份。两种方式：

1. **后台贴链接**（推荐）：在「身份管理」顶部「AI 生成身份」框贴链接，自动采集 + AI 生成身份。
2. **小号发消息**：用小号给你的店铺发「**【采集】** + 商品链接」，Worker 会自动抓取入库并回执一条采集结果。

> 采集是**复用 Worker 那个已登录的 Edge** 打开商品页抓的，所以**必须 Worker 在线、Edge 已登录闲鱼**，否则抓不到（会提示"未登录或被风控"）。

## 七、核心机制

这些是后台看不到、但一直在保护你的：

- **不回错人**：发送前先切到对应买家的对话框，再**回读该框的买家 ID 确认无误才发**；多条消息同时来时**串行处理**（一次只发一个），杜绝群发错乱。
- **不串台**：每个会话独立记忆；万一抓不到会话 ID，宁可**不回**也不冒险回错（`require_cid`）。
- **身份会话锁定**：身份一旦确定，整段对话沿用，不中途跳变。
- **去重**：每条消息有唯一 ID 并入库，Worker 重启也不会把旧消息重复回一遍。
- **拟人延迟**：回复不是秒回，而是"基础思考时间 + 随机扰动 + 轻微长度影响"（约 2.5~16 秒），更像真人。可在 config 调。
- **单实例保护**：Worker 用端口锁保证全局只有一个在跑，多余的会自动退出——**防止重复回消息**。
- **健康自动重启**：Worker 掉线会自动拉回来。

## 八、config.yml 参数说明

文件在项目根目录 `config.yml`（含密钥，已被 git 忽略）。**改完要重启后端生效**。

| 配置 | 说明 |
|---|---|
| `ai.api_key` | DeepSeek 密钥（必填） |
| `ai.model` | 模型，默认 `deepseek-chat` |
| `ai.enabled` | AI 总开关 |
| `reply_delay.*` | 回复延迟参数。想更快就调小 `base_min/base_max` 和 `min_delay` |
| `safety.daily_reply_limit` | 每日回复上限 |
| `safety.business_hours` | 营业时段，如 `[0, 24]` 表示全天；`[8, 23]` 表示 8 点到 23 点才回 |
| `safety.require_cid` | true=抓不到会话 ID 就不回（防串台，推荐 true） |
| `safety.route_fallback` | 找不到买家会话时：`active`/`last_buyer`/`none` |
| `fallback_reply` | AI 失败时的兜底话术 |

## 九、⚠️ 注意事项与常见问题（重要）

1. **改了后端代码（app 里的 .py）或 config.yml，必须重启**才生效——`stop.bat` 再 `start.bat`。只改网页界面则 Ctrl+F5 强刷即可。

2. **「上线运行」会关掉你当前所有 Edge 窗口**，再开一个专用的调试 Edge——这是接管浏览器方案的固有代价。所以建议：跑自动化时，用那个专用 Edge，别再额外开 Edge 窗口。

3. **不要手动多开 Worker**，也不要既用 `start.bat` 又自己命令行起——虽然有单实例锁兜底，但统一用 `start.bat` 最省心。

4. **采集 / AI 生成身份失败、提示"未登录或被风控"**：去那个调试 Edge 里确认闲鱼是登录状态。

5. **回复日志里出现 `fallback`**：偶尔一两条是 DeepSeek 网络波动，正常；频繁出现去看 `logs/worker.log` 里的 `AI 回复失败` 后面跟的具体原因。

6. **AI 偶尔答得不好 / 抽取需求有误**：去对应「身份管理」改人设/目标/槽位，或在「客户记忆」里手动改摘要；改完先去「AI 试聊」验证。

7. **第一次用某个商品**：建议先用「AI 生成身份」贴链接生成身份并保存，或手动建好身份 + 报价策略，再让它自动回复。

8. **关键信息别乱承诺**：AI 是辅助，涉及大额、号的安全、交付细节等，建议「暂停 AI」自己把关。

9. **数据都在哪**：`data/xianyu_v2.db`（SQLite）。身份、客户记忆、日志、报价策略、采集商品都在这。备份就备它。

## 附：日常推荐流程

1. 双击 `start.bat` → 后台自动打开、Worker 自动上线。
2. 新商品：身份管理「AI 生成身份」贴链接 → 保存；需要的话配下报价策略。
3. 「AI 试聊」验证身份/话术效果。
4. 平时盯「客户记忆」（看未读、需求文档）和「回复日志」。
5. 重要客户「暂停 AI」自己聊。
6. 收工双击 `stop.bat`。
